[문제 2 - 15퍼즐]
## 기능 요구사항
- 타이틀과 현재 턴을 표시한다.
- 다음 줄부터 1 - 15까지의 숫자와 [ ] 모양의 공백칸 하나를 잘 섞어서 배치한다.
- 마지막 줄에 숫자 입력> 프롬프트를 출력한다.
- 사용자로부터 1-15까지의 숫자를 입력받는다.
- 입력에 문제가 있을 경우 다시 입력을 받는다.
- 정상적으로 입력을 받았을 경우 숫자를 [ ] 위치로 밀어낸다.
- 모든 숫자가 올바르게 정렬되었을 경우 축하메시지와 턴을 표시한다.

## 구현 과정
1단계에서의 시행착오를 바탕으로 숫자 퍼즐의 상태를 조회하거나 바꿀 수 있는 Numbers 클래스를 만들고, 요구사항에 맞게 멤버 변수와 메서드를 구현했습니다.
해당 문제는 다음의 네 단계를 거치며 하나씩 해결했습니다.

1. 객체 생성
- 초기 구상 : 생성자를 정의하는 코드에서 1 ~ 15 까지 랜덤한 정수를 이차원 배열에 무작위로 할당하는 로직을 추가하려고 했습니다.
- 문제점 : 하지만 특정 조건에서 절대 풀리지 않는 15 퍼즐이 존재할 수 있다는 문제를 발견했습니다.
- 해결 : 정적 팩토리 메서드 패턴을 참고하여 생성자 코드가 생성 외의 책임을 지지 않고, createNumbers()와 checkNumbers() 메서드를 통해 랜덤 배열 생성 및 퍼즐 유효성 검증의 책임을 분리했습니다.
- 결과 : 생성자의 책임을 줄이면서, 풀릴 수 있는 15 퍼즐을 정적 메서드 createNumbers()의 반환 객체로 간단하게 얻을 수 있습니다.

2. 입력 유효성 검증
- main에 선언된 checkInput() static 메서드는 사용자의 입력이 정수가 맞는지, 범위는 1 ~ 15 안에 있는지 검사합니다.
- 조건에 해당하지 않는다면 -1를 반환하여 해당 Turn을 continue로 뛰어넘고 다시 사용자의 입력을 받도록 만들었습니다.

3. 빈칸과 입력한 값의 위치 변경
- 빈칸(0)의 좌표와 입력된 값의 좌표를 바꾸기 위해 먼저 빈칸의 좌표 주변(상/하/좌/우)에 입력된 값이 존재하는지 검증했습니다(isNeighbor).
- 해당 조건을 통과하면 getCoordinate() 메서드를 통해 구한 입력값의 기존 좌표의 값과 빈칸의 기존 좌표의 값을 바꿨습니다.
- 마지막으로 새로운 빈칸 좌표를 갱신하고 true를 반환합니다.

4. 정렬 검증
- swap 로직이 끝난 뒤, 변경된 15 퍼즐이 오름차순으로 정렬되어 있는지 검사합니다.
- 1 ~ 15 까지 순서대로 정렬되어 있고 마지막 요소에 빈칸으로 처리되는 0이 있다면 게임 통과로 간주하고 반복문을 빠져나옵니다.
- 통과하지 못하면 다음 Turn을 출력하고 앞선 과정을 재반복합니다.
